# Ember 授权申请

本仓库用于 Ember 授权申请与授权文件签发。

## 授权流程

1. 打开 Ember 的“授权”页面，复制当前机器码。
2. 使用自己的 Gitee 账号，在任意公开 Gitee issue 中评论该机器码。
3. 回到本仓库，进入 `Issues`，点击 `New issue`。
4. 选择 `Ember 授权申请`。
5. 填写机器码、Gitee 用户 ID、Gitee 评论链接。
6. 创建 issue 后，GitHub Action 会自动校验 Gitee 评论并签发授权。
7. 签发成功后，回到 Ember 授权页等待自动刷新，或者手动点击“刷新授权”。

## 说明

- 一个 Gitee 账号同一时间只绑定一个机器码。
- 如果更换电脑或机器码变化，重新提交授权申请即可自动换绑到最新机器。
- Gitee 评论链接必须是公开可访问的 issue 评论。
- GitHub issue 创建后不需要手动下载文件，软件会自动尝试刷新授权。
- 如果网络不好，也可以在签发成功后手动下载对应 `.lic` 文件，再到 Ember 授权页选择导入。

## 常见问题

### 为什么需要 Gitee 评论？

Gitee 账号通常需要手机号验证，可以增加账号滥用和倒卖成本。

### 授权失败怎么办？

请检查：

- 机器码是否完整复制。
- Gitee 用户 ID 是否填写正确。
- Gitee 评论链接是否公开可访问。
- Gitee 评论内容是否包含同一个机器码。

修改后可以编辑 issue 内容重新触发授权。
